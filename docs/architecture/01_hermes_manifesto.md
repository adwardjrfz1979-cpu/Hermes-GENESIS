===============================================================================
Why Hermes Exists
===============================================================================

Software has become fragmented.

Knowledge lives in one application.
Memory in another.
Automation somewhere else.
Artificial Intelligence somewhere else again.

Modern AI systems are becoming increasingly capable, but most remain tightly
coupled to a particular model, provider, interface, or cloud platform.

Hermes exists to solve a different problem.

Hermes is not built around today's models.

Hermes is built around the belief that intelligence itself should be modular,
replaceable, observable, and capable of evolving without redesigning the system
that hosts it.

The architecture should outlive the technologies it orchestrates.

===============================================================================
Vision
===============================================================================

Hermes aims to become an Intelligence Runtime.

Not another chatbot.

Not another AI wrapper.

Not another automation tool.

Hermes should provide a stable environment where reasoning, memory,
knowledge, automation, planning, and future AI systems can work together
regardless of which models, providers, or protocols exist.

If the best model changes tomorrow, Hermes should adapt.

If an entire provider disappears, Hermes should continue operating.

The system should evolve while preserving its architecture.

===============================================================================
What Hermes Is
===============================================================================

Hermes is:

• an Intelligence Runtime

• an orchestration engine

• a capability platform

• a reasoning environment

• an execution runtime

• a knowledge system

• a memory system

• an automation platform

Hermes is not defined by any individual model.

===============================================================================
What Hermes Is Not
===============================================================================

Hermes is not:

• an OpenAI application

• a chatbot with plugins

• an API collection

• a prompt library

• an automation script

• a vendor-specific assistant

Every external dependency must remain replaceable.

===============================================================================
Core Philosophy
===============================================================================

Hermes separates responsibilities.

Intent identifies what the user wants.

Planning decides how to achieve it.

Reasoning evaluates options.

Capabilities perform work.

Protocols connect systems.

Execution performs actions.

Observation records outcomes.

Reflection evaluates decisions.

Memory decides what deserves to be remembered.

Every subsystem owns exactly one responsibility.

===============================================================================
Architectural Laws
===============================================================================

Law 1

Everything is replaceable.

Models.

Providers.

Protocols.

Databases.

User interfaces.

Storage engines.

Nothing is permanent.

-------------------------------------------------------------------------------

Law 2

Capabilities matter more than models.

Hermes routes based on what is required,
not who provides it.

-------------------------------------------------------------------------------

Law 3

Reason before execution.

No autonomous action should occur without planning.

-------------------------------------------------------------------------------

Law 4

Memory is earned.

Remembering everything is not intelligence.

Remembering the right things is.

-------------------------------------------------------------------------------

Law 5

Truth over agreement.

Hermes values accuracy over approval.

It acknowledges uncertainty.

It asks questions when required.

It never pretends confidence it does not possess.

-------------------------------------------------------------------------------

Law 6

Restraint is intelligence.

The smartest action is often:

asking one question,

using one tool,

saving no memory,

doing nothing.

More output does not mean better intelligence.

-------------------------------------------------------------------------------

Law 7

The human remains in control.

Automation exists to reduce effort,
never to remove meaningful human choice.

-------------------------------------------------------------------------------

Law 8

Complexity belongs inside Hermes,
not inside the user's workflow.

Whenever practical,
configuration should move from source code
to understandable interfaces.

-------------------------------------------------------------------------------

Law 9

Observe before optimizing.

Evidence comes before optimization.

Metrics come before assumptions.

-------------------------------------------------------------------------------

Law 10

Every subsystem should evolve independently.

Adding a model,
provider,
protocol,
or capability
should require minimal architectural change.

===============================================================================
Intelligence Philosophy
===============================================================================

Hermes does not measure intelligence by the amount of text it produces.

Intelligence is judgment.

Judgment means:

knowing when to think,

knowing when to search,

knowing when to ask,

knowing when to execute,

knowing when to remember,

and knowing when to remain silent.

Generation is only one capability among many.

===============================================================================
User Philosophy
===============================================================================

The user remains the owner of their system.

Memory should be transparent.

Actions should be explainable.

Knowledge should be portable.

Data should never become trapped inside one provider.

Users should always know:

what was remembered,

what tool was used,

what model responded,

and why a decision was made.

Hermes exists to increase human capability,
not dependence.

===============================================================================
What Hermes Refuses To Become
===============================================================================

Hermes will never optimize for engagement.

Hermes will never intentionally manipulate users.

Hermes will never manufacture urgency.

Hermes will never ask unnecessary follow-up questions simply to keep a
conversation going.

Hermes will never fabricate actions.

Hermes will never fabricate sources.

Hermes will never hide uncertainty.

Hermes will never silently create permanent memory.

Hermes will never sacrifice architecture for convenience.

Hermes will never become dependent on a single provider.

===============================================================================
Success Criteria
===============================================================================

Hermes succeeds when:

new models integrate without architectural changes.

new capabilities require minimal code.

users trust its decisions.

actions remain observable.

automation remains controllable.

memory remains useful.

the architecture survives technological change.

Hermes is successful not because it always knows the answer.

Hermes is successful because it consistently exercises good judgment.

===============================================================================
Final Principle
===============================================================================

Technology changes.

Models improve.

Protocols evolve.

Hardware advances.

Architecture should endure.

The purpose of Hermes is not to build around today's intelligence.

The purpose of Hermes is to build the environment in which tomorrow's
intelligence can exist.
